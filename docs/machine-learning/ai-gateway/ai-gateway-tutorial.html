Tutorial: Create routes and query OpenAI provider models using Databricks AI Gateway

Preview

This feature is in Private Preview. To try it, reach out to your Databricks contact.

Warning

The guidance in this article is for AI Gateway in Private Preview. Databricks recommends using Model Serving support of external models, a Public Preview capability that allows you to centrally manage and access third party hosted foundation models.

This article provides step-by-step instructions to configure and query three AI Gateway routes to OpenAI services for completions, chat, and embeddings. The steps in this guide can be adapted for other AI Gateway providers.

For additional information about Databricks AI Gateway and its corresponding terminology, see Databricks AI Gateway.

Requirements

  • To enroll in the Private Preview, please submit the enrollment form.

  • Databricks Runtime 13.0 ML and above.

  • MLflow 2.5 and above.

  • OpenAI API key.

Install the Databricks AI Gateway client

To install the Databricks AI Gateway Python client, you can use %pip in your Databricks notebook as follows:

%pip install 'mlflow[gateway]'

Set the OpenAI API key(s) for each provider

In order for the AI Gateway service to communicate with the OpenAI API, you need an API key. You can create an API key from the OpenAI dashboard.

For this example, you only connect with OpenAI. For additional providers within the configuration, those corresponding API keys need to be set as well.

Once you have the key, Databricks recommends storing it using Databricks Secrets. For this guide, you can assume that the OpenAI key is available in secret scope example with key openai-api-key.

Set the AI Gateway URI

If you are using the AI Gateway in a Databricks notebook or Databricks job, you can set the gateway URI as follows:

from mlflow.gateway import set_gateway_uri

set_gateway_uri(gateway_uri="databricks")

If you are using the AI Gateway outside of a Databricks notebook or Databricks job, you need to configure your Databricks host name and personal access token in your current Python environment before making requests to the AI Gateway. You can do this using the DATABRICKS_HOST and DATABRICKS_TOKEN environment variables.

For example:

import os
from mlflow.gateway import set_gateway_uri

os.environ["DATABRICKS_HOST"] = "http://your.workspace.databricks.com"
os.environ["DATABRICKS_TOKEN"] = "<your_personal_access_token>"

set_gateway_uri(gateway_uri="databricks")

Create AI Gateway routes

You can create AI Gateway routes for each large language model (LLM) you want to use with the mlflow.gateway.create_route() API.

For more information, see the Fluent API and the Client API.

After you set the AI Gateway URI in your Python environment, you can create separate routes for completions, chat, and embeddings as follows:

from mlflow.gateway import create_route

openai_api_key = dbutils.secrets.get(
    scope="example",
    key="openai-api-key"
)

# Create a route for completions with OpenAI GPT-4
create_route(
    name="completions",
    route_type="llm/v1/completions",
    model={
        "name": "gpt-4",
        "provider": "openai",
        "openai_config": {
            "openai_api_key": openai_api_key
        }
    }
)

# Create a route for chat with OpenAI GPT-4
create_route(
    name="chat",
    route_type="llm/v1/chat",
    model={
        "name": "gpt-4",
        "provider": "openai",
        "openai_config": {
            "openai_api_key": openai_api_key
        }
    }
)

# Create a Route for embeddings with OpenAI text-embedding-ada-002
create_route(
    name="embeddings",
    route_type="llm/v1/embeddings",
    model={
        "name": "text-embedding-ada-002",
        "provider": "openai",
        "openai_config": {
            "openai_api_key": openai_api_key
        }
    }
)

If you no longer need a route, you can delete it using the mlflow.gateway.delete_route API.

Send requests using the Fluent API

The next step is to query the routes using the Fluent API.

For information on formatting requirements, how to pass parameters and how to send requests using other MLflow Gateway APIs, see Query models made accessible using the AI Gateway.

Example: Send a completions request

The following is an example of how to send a completions request using the Fluent API:

from mlflow.gateway import set_gateway_uri, query

set_gateway_uri("databricks")

response = query(
    route="completions",
    data={"prompt": "What is the best day of the week?", "temperature": 0.3}
)

print(response)

The returned response has the following structure:

{
     "candidates": [
       {
         "text": "It's hard to say what the best day of the week is.",
         "metadata": {
           "finish_reason": "stop"
         }
       }
    ],
    "metadata": {
        "input_tokens": 13,
        "output_tokens": 15,
        "total_tokens": 28,
        "model": "gpt-4",
        "route_type": "llm/v1/completions"
    }
}

Example: Send a chat request

The following is an example of how to send a chat request using the Fluent API:

from mlflow.gateway import set_gateway_uri, query

set_gateway_uri("databricks")

response = query(
    route="chat",
    data={"messages": [{"role": "user", "content": "What is the best day of the week?"}]}
)

print(response)

The returned response has the following structure:

{
    "candidates": [
        {
            "message": {
                "role": "assistant",
                "content": "\n\nIt's hard to say what the best day of the week is.",
            },
            "metadata": {"finish_reason": "stop"}
        }
    ],
    "metadata": {
        "input_tokens": 13,
        "output_tokens": 15,
        "total_tokens": 28,
        "model": "gpt-4",
        "route_type": "llm/v1/completions"
    }
}

Example: Send an embeddings request

The following is an example of how to send an embeddings request using the Fluent API:

from mlflow.gateway import set_gateway_uri, query

set_gateway_uri("databricks")

response = query(
    route="embeddings",
    data={"text": ["Example text to embed"]}
)

print(response)

The returned response has the following structure, content and token values vary:

{
    "embeddings": [
      0.010169279,
      -0.0053696977,
      -0.018654726,
      -0.03396831,
      3.1851505e-05,
      -0.03341145,
      -0.023189139,
      ...
    ],
    "metadata": {
        "input_tokens": 6,
        "total_tokens": 6,
        "model": "text-embedding-ada-002",
        "route_type": "llm/v1/embeddings"
    }
}

Compare provider models

The following is an example of adding and querying a new model from a different provider, Anthropic, to determine which model is better for a given use case. You can assume that the Anthropic API key is stored in Databricks Secrets with scope example and key anthropic-api-key.

from mlflow.gateway import set_gateway_uri, create_route, query

set_gateway_uri("databricks")

anthropic_api_key = dbutils.secrets.get(
    scope="example",
    key="anthropic-api-key"
)

# Create a Route for completions with OpenAI GPT-4
create_route(
    name="claude-completions",
    route_type="llm/v1/completions",
    model={
        "name": "claude-v1.3",
        "provider": "anthropic",
        "anthropic_config": {
            "anthropic_api_key": anthropic_api_key
        }
    }
)

completions_response = query(
    route="claude-completions",
    data={"prompt": "What is MLflow? Be concise.", "temperature": 0.3}
)

The returned response has the following structure:

{
    "candidates": [
        {
            "text": "MLflow is an open source platform for machine learning...",
            "metadata": {
                "finish_reason": "stop"
            }
        }
    ],
    "metadata": {
        "input_tokens": 8,
        "output_tokens": 15,
        "total_tokens": 23,
        "model": "claude-v1.3",
        "route_type": "llm/v1/completions"
    }
}
