Enable streaming output in Databricks AI gateway

Preview

This feature is in Private Preview. To try it, reach out to your Databricks contact.

Warning

The guidance in this article is for AI Gateway in Private Preview. Databricks recommends you migrate to the public preview experience called External Models.

When making a scoring request, you can set the stream parameter to incrementally stream the response using sever-sent events (SSE). Instead of waiting for the entire response to be generated, you can access results as it is generated, making it ideal for applications like chat bots that require immediate responses.

The following providers support streaming:

Provider

Route types

Anthropic

completions

Cohere

completions

OpenAI

chat, completions

Learn more about Databricks AI Gateway.

Chat request example

The following is a chat request example

curl \
  $DATABRICKS_HOST/gateway/$ROUTE/invocations \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $DATABRICKS_TOKEN" \
  -d '{
    "messages": [
      {
        "role": "user",
        "content": "What is MLflow?"
      }
    ],
    "stream": true
  }'


import requests
import os

DATABRICKS_HOST= os.environ["DATABRICKS_HOST"]
DATABRICKS_TOKEN= os.environ["DATABRICKS_TOKEN"]
ROUTE= os.environ["ROUTE"]

resp = requests.post(
    f"{DATABRICKS_HOST}/gateway/{ROUTE}/invocations",
    headers={"Authorization": f"Bearer {DATABRICKS_TOKEN}"},
    json={
        "messages": [{"role": "user", "content": "What is MLflow?"}],
        "stream": True,
    },
    stream=True,
)
resp.raise_for_status()

for l in resp.iter_lines():
    print(l.decode("utf-8"))

async function main() {
  const { DATABRICKS_HOST, DATABRICKS_TOKEN, ROUTE } = process.env;
  const response = await fetch(
    `${DATABRICKS_HOST}/gateway/${ROUTE}/invocations`,
    {
      method: "POST",
      body: JSON.stringify({
        messages: [{ role: "user", content: "What is MLflow?" }],
        stream: true,
      }),
      headers: {
        Authorization: `Bearer ${DATABRICKS_TOKEN}`,
      },
    }
  );
  const reader = response.body.getReader();
  const decoder = new TextDecoder("utf-8");
  while (true) {
    const { value, done } = await reader.read();
    if (done) break;
    console.log(decoder.decode(value));
  }
}

main();

Chat response format and example

The following shows the query response format, where role, content and finish_reason are optional parameters.

{
  "candidates": [
    {
      "index": integer,
      "message": {
        "role": string, # optional
        "content": string # optional
      },
      "metadata": {
        "finish_reason": string # optional
      }
    }
  ],
  "metadata": {
    "model": string,
    "route_type": string
  }
}

The following is a chat response example:


event: output
data: {"candidates":[{"index":0,"message":{"role":"assistant","content":""},"metadata":{}}],"metadata":{"model":"gpt-3.5-turbo-0613","route_type":"llm/v1/chat"}}

event: output
data: {"candidates":[{"index":0,"message":{"content":"an"},"metadata":{}}],"metadata":{"model":"gpt-3.5-turbo-0613","route_type":"llm/v1/chat"}}

...

event: output
data: {"candidates":[{"index":0,"message":{"content":"."},"metadata":{}}],"metadata":{"model":"gpt-3.5-turbo-0613","route_type":"llm/v1/chat"}}

event: output
data: {"candidates":[{"index":0,"message":{},"metadata":{"finish_reason":"stop"}}],"metadata":{"model":"gpt-3.5-turbo-0613","route_type":"llm/v1/chat"}}

event: done
data: {}
