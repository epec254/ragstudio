Use MLflow to query serving endpoints

Important

The MLflow Deployments predict() API is Experimental and the API definition might change.

In this article, you learn how to format scoring requests for a served model, and how to use the MLflow Deployments API to send those requests to the model serving endpoint. This article includes examples for querying,

  • Foundation models, which include:

    • Third-party foundation models, referred to as external models.

    • State-of-the-art open models made available by Foundation Model APIs.

    • Curated foundation models on Databricks Marketplace, an open marketplace for sharing third-party data and AI assets.

  • Custom models. MLflow models that are installed in Unity Catalog or registered in the workspace model registry.

Databricks Model Serving provides standardized query interfaces for serving custom and foundation models and follows the OpenAI interface for foundation models.

The MLflow Deployments client provides the predict() API for sending scoring requests. The predict() API accepts the same parameters as the REST API for serving endpoints. See POST /serving-endpoints/{name}/invocations in the API reference.

You can also query endpoints using the REST API.

Requirements

  • A model serving endpoint.

  • Databricks Runtime 13.0 ML or above.

  • MLflow 2.9 or above.

  • The MLflow Deployment client. To install it, run:

    import mlflow.deployments

client = mlflow.deployments.get_deploy_client("databricks")

Scoring request for a custom model

The following is an example request for a custom model.


client = mlflow.deployments.get_deploy_client("databricks")

response = client.predict(
            endpoint="test-model-endpoint",
            inputs={"dataframe_split": [{
                    "index": [0, 1],
                    "columns": ["sepal length (cm)", "sepal width (cm)", "petal length (cm)", "petal width (cm)"],
                    "data": [[5.1, 3.5, 1.4, 0.2], [4.9, 3.0, 1.4, 0.2]]
                    }]
                }
           )

Scoring request for foundation models

This section shows example scoring requests and responses for foundation models.

MLflow Deployments uses the same parameters as the REST API. See POST /serving-endpoints/{name}/invocations in the API reference for standard query parameters.

For external models (models hosted outside of Databricks) you can include additional parameters that are valid for a given provider and endpoint configuration. See Additional query parameters.

For Foundation Model APIs supported models, you can also use the Experimental Foundation Model APIs Python SDK for querying those endpoints. See How to query Foundation Model APIs with the Python SDK.

Example: Send a chat request

The following sends a request to gpt-3.5-turbo hosted by OpenAI.

import mlflow.deployments

client = mlflow.deployments.get_deploy_client("databricks")
chat_response = client.predict(
    endpoint="openai-chat-endpoint",
    inputs={
        "messages": [
            {
              "role": "user",
              "content": "Hello!"
            },
            {
              "role": "assistant",
              "content": "Hello! How can I assist you today?"
            },
            {
              "role": "user",
              "content": "Can you explain AI in ten words?"
            }
        ],
        "temperature": 0.1,
        "max_tokens": 20
    }
)
chat_response == {
    "id": "chatcmpl-8QViByRsjmKLm7KTBKMrJzstE01oH",
    "object": "chat.completion",
    "created": 1701329119,
    "model": "gpt-3.5-turbo",
    "choices": [
        {
            "index": 0,
            "finish_reason": "stop",
            "message": {
                "role": "assistant",
                "content": "Artificial intelligence: mimicking human intelligence through"
                           " advanced computing and algorithms."
            }
        }
    ],
    "usage": {
        "prompt_tokens": 34,
        "completion_tokens": 25,
        "total_tokens": 59
    }
}

Example: Send a completions request

The following sends a request to gpt-3.5-turbo-instruct hosted by OpenAI.

completions_response = client.predict(
    endpoint="openai-completions-endpoint",
    inputs={
        "prompt": "What is the capital of France?",
        "temperature": 0.1,
        "max_tokens": 10,
        "n": 2
    }
)
completions_response == {
    "id": "cmpl-8QW0hdtUesKmhB3a1Vel6X25j2MDJ",
    "object": "text_completion",
    "created": 1701330267,
    "model": "gpt-3.5-turbo-instruct",
    "choices": [
        {
            "text": "The capital of France is Paris.",
            "index": 0,
            "finish_reason": "stop",
            "logprobs": None
        },
        {
            "text": "Paris is the capital of France",
            "index": 1,
            "finish_reason": "stop",
            "logprobs": None
        },
    ],
    "usage": {
        "prompt_tokens": 7,
        "completion_tokens": 16,
        "total_tokens": 23
    }
}

Example: Send an embeddings request

The following sends a request to text-embedding-ada-002 hosted by OpenAI.

embeddings_response = client.predict(
    endpoint="openai-embeddings-endpoint",
    inputs={
        "input": "Here is some text to embed"
    }
)
embeddings_response == {
    "object": "list",
    "data": [
        {
            "object": "embedding",
            "index": 0,
            "embedding": [

                0.007892851,

                -0.014019844
            ]
        }
    ],
    "model": "text-embedding-ada-002",

}
