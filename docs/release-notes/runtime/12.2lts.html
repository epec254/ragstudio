Databricks Runtime 12.2 LTS

The following release notes provide information about Databricks Runtime 12.2 LTS, powered by Apache Spark 3.3.2.

Databricks released these images in March, 2023.

New features and improvements

  • Delta Lake schema evolution supports specifying source columns in merge statements

  • Structured Streaming workloads are supported on clusters with shared access mode

  • New features for Predictive I/O

  • Implicit lateral column aliasing support

  • New forEachBatch feature

  • Standardized Connection Options for Query Federation

  • Extended SQL function library for array management

  • New mask function to anonymize strings

  • Common error conditions now return SQLSTATEs

  • Invoke generator functions in the FROM clause

  • Support for protocol buffers is generally available

  • Bug fixes

Delta Lake schema evolution supports specifying source columns in merge statements

You can now specify columns present only in the source table in insert or update actions for merge statements when schema evolution is enabled. In Databricks Runtime 12.1 and below, only INSERT * or UPDATE SET * actions can be used for schema evolution with merge. See Automatic schema evolution for Delta Lake merge.

Structured Streaming workloads are supported on clusters with shared access mode

You can now use Structured Streaming to interact with Unity Catalog on shared clusters. Some limitations apply. See What Structured Streaming functionality does Unity Catalog support?.

New features for Predictive I/O

Photon support for Foreachbatch sink is now available. Workloads that stream from a source and merge into Delta Tables or write to multiple sinks can now benefit from Photonized Foreachbatch sink.

Implicit lateral column aliasing support

Databricks now supports implicit lateral column aliasing by default. You can now reuse an expression specified earlier in the same SELECT list. For example, given SELECT 1 AS a, a + 1 AS b, the a in a + 1 can be resolved as the previously defined 1 AS a. Check Name resolution for more details in resolution order. To turn off this feature, you can set spark.sql.lateralColumnAlias.enableImplicitResolution to false.

New forEachBatch feature

Photon is now supported when using foreachBatch to write to a data sink.

Standardized Connection Options for Query Federation

You can now use a unified set of options (host, port, database, user, password)to connect to data sources supported in Query Federation. Port is optional and uses the default port number for each data source, unless specified.

Extended SQL function library for array management

You can now remove all NULL elements from an array using array_compact. To append elements to an array, use array_append.

New mask function to anonymize strings

Invoke the mask function to anonymize sensitive string values.

Common error conditions now return SQLSTATEs

Most error conditions that Databricks Runtime now include documented SQLSTATE values which can be used to test for errors in a SQL standard compliant way.

Invoke generator functions in the FROM clause

You can now invoke table-valued generator functions such as explode in the regular FROM clause of a query. This aligns generator function invocation with other built-in and user defined table functions.

Support for protocol buffers is generally available

You can use the from_protobuf and to_protobuf functions to exchange data between binary and struct types. See Read and write protocol buffers.

Bug fixes

  • Improved consistency for Delta commit behavior for empty transactions relating to update, delete, and merge commands. In WriteSerializable isolation level, commands that result in no changes now create an empty commit. In a Serializable isolation level, such empty transactions now do not create a commit.

Behavior changes

Behavior changes with the new lateral column alias feature

The new lateral column alias feature introduces behavior changes for the following cases during name resolution:

  • Lateral column alias now takes precedence over correlated references with the same name. For example, for this query SELECT (SELECT c2 FROM (SELECT 1 AS c1, c1 AS c2) WHERE c2 > 5) FROM VALUES(6) AS t(c1), the c1 in the inner c1 AS c2 was resolved to the correlated reference t.c1, but now changes to lateral column alias 1 AS c1. The query now returns NULL.

  • Lateral column alias now takes precedence over function parameters with the same name. For example, for the function, CREATE OR REPLACE TEMPORARY FUNCTION func(x INT) RETURNS TABLE (a INT, b INT, c DOUBLE) RETURN SELECT x + 1 AS x, x, the x in the function body was resolved to the function parameter x, but changes to lateral column alias x + 1 in the function body. The query, SELECT * FROM func(1) now returns 2, 2.

  • To turn off the lateral column alias feature, set spark.sql.lateralColumnAlias.enableImplicitResolution to false. For more information, see Name resolution.

Library upgrades

  • Upgraded Python libraries:

    • filelock from 3.8.2 to 3.9.0

    • joblib from 1.1.0 to 1.1.1

    • platformdirs from 2.6.0 to 2.6.2

    • whatthepatch from 1.0.3 to 1.0.4

  • Upgraded R libraries:

    • class from 7.3-20 to 7.3-21

    • codetools from 0.2-18 to 0.2-19

    • MASS from 7.3-58 to 7.3-58.2

    • nlme from 3.1-160 to 3.1-162

    • Rserve from 1.8-11 to 1.8-12

    • SparkR from 3.3.1 to 3.3.2

Behavior changes

  • Users are now required to have SELECT and MODIFY privileges on any file when creating a schema with a defined location.
