Databricks Feature Engineering in Unity Catalog client and Databricks Feature Store client release notes

This page lists releases of the Databricks Feature Engineering in Unity Catalog client and the Databricks Feature Store client, available on PyPI as databricks-feature-engineering and databricks-feature-store.

The libraries are used to:

  • Create, read, and write feature tables

  • Train models on feature data

  • Publish feature tables to online stores for real-time serving

For usage documentation, see Databricks Feature Store. For Python API documentation, see Python API.

The Feature Engineering in Unity Catalog client works for Feature Engineering in Unity Catalog and requires manual installation (pip install databricks-feature-engineering).

The Feature Store client works for Workspace Feature Store and is packaged with Databricks Runtime for Machine Learning. It can also run on Databricks Runtime after installing it from PyPI (pip install databricks-feature-store). For unit testing only, both clients can be used locally or in CI/CD environments.

For a table showing which databricks-feature-engineering and databricks-feature-store releases are compatible with which Databricks Runtime and Databricks Runtime ML versions, see Feature Engineering compatibility matrix.

databricks-feature-store 0.14.0

Starting with 0.14.0, you must specify timestamp key columns in the primary_keys argument. Timestamp keys are part of the “primary keys” that uniquely identify each row in the feature table. Like other primary key columns, timestamp key columns cannot contain NULL values.

In the following example, the DataFrame user_features_df contains the following columns: user_id, ts, purchases_30d, and is_free_trial_active.

0.14.0 and above 

fs = FeatureStoreClient()

fs.create_table(
name="ads_team.user_features",
primary_keys=["user_id", "ts"],
timestamp_keys="ts",
features_df=user_features_df,
)

0.13.1 and below 

fs = FeatureStoreClient()

fs.create_table(
name="ads_team.user_features",
primary_keys="user_id",
timestamp_keys="ts",
features_df=user_features_df,
)
