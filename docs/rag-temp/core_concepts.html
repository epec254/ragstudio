RAG Application

A RAG Application is the container for a single RAG use case. It contains all code and configuration for the RAG application and is managed as a Databricks Asset Bundles (DABs).

A RAG Application consists of:

  1. Data Ingestors

  2. RAG Versions

  3. Metric Configurations

Data Ingestor

A Data Ingestor lands raw unstructured documents into a Volume, which are Unity Catalog objects representing a logical volume of storage in a cloud object storage location. A Data Ingestor consists of a:

  1. Raw Data Source: The original source of the documents, for example, a Google Drive Folder, Sharepoint Library, or Confluence Wiki.

  2. Ingested Data Store: The UC Volume where the raw, ingested documents are stored

  3. Ingestion Pipeline: A pipeline that loads the raw documents from the Raw Data Source into the Ingested Data Store.

Each RAG application has 1 or more Data Ingestors.
