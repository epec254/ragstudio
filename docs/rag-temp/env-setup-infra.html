Infrastructure setup

This document walks you through configuring the infrastructure that is used by a RAG Studio application.

(Required) Databricks Workspace

A RAG Studio app must deployed within a Databricks Workspace.

  1. Select a Databricks Workspace with Unity Catalog and Serverless enabled in a supported region. Note the URL of the Workspace to use when configuring the application e.g., https://workspace-name.cloud.databricks.com.

(Required) Unity Catalog

Many RAG Studio assets are created within a Unity Catalog schema.

  1. Do one of the following to select a Unity Catalog schema to use for your RAG Application.

    • Create a new catalog / schema

      1. Create a new catalog and new schema inside the catalog.

      2. Create a new schema inside an existing catalog.

    • Select an existing catalog / schema

      1. Select an existing Unity Catalog catalog and schema. You can view existing catalogs and schemas by going to https://workspace-name.cloud.databricks.com/explore/data

  2. Assign the correct permissions to the catalog / schema

(Required) Generative AI Models

RAG Studio natively integrates with Databricks Model Serving for access to Foundational Models. This integeration is required to use RAG Studioâ€™s LLM Judged Assessments, but is optional for using Foundational Models from within a RAG application.

  1. Option 1 use Open Source models hosted by Databricks Foundation Model APIs with pay-per-token.

    1. No additional set up is required to use LLaMa2-70B-Chat and BGE-Large-EN

  2. Option 2 use Open Source models hosted by Databricks Foundation Model APIs with provisioned throughput.

    1. Follow the steps for deploying a provisioned throughput model to load any supported open source model with a llm/v1/embeddings or llm/v1/chat

  3. Option 3 use an External Model such as (Azure) OpenAI.

    To use OpenAI or Azure OpenAI, you need to add your OpenAI secrets using a External models in Databricks Model Serving:

    1. Configure a llm/v1/embeddings model - (Azure) OpenAI text-embeddings-ada-002

    2. Configure a llm/v1/chat model - either (Azure) OpenAI gpt-35-turbo or gpt-4

    Similar steps can be followed for other model providers.

(Optional) Vector Search

RAG Studio natively intergates with Databricks Vector Search, automatically creating index versions that correspond to versions of your application. Vector Search indexes contain the chunked and embedded unstructured documents that power your application.

RAG Studio can automatically create a Vector Search Endpoint, or you can choose create an endpoint yourself or use an existing endpoint:

  1. Do one of the following:

    1. Create a new Vector Search Endpoint using the UI or Python SDK

    2. Select an existing Vector Search endpoint. You can view existing endpoints in your workspace by going to https://workspace-name.cloud.databricks.com/#setting/clusters/vector-search
