Databricks Runtime 5.4 (unsupported)

Databricks released this image in June 2019.

The following release notes provide information about Databricks Runtime 5.4, powered by Apache Spark.

New Features

Delta Lake on Databricks adds Auto Optimize (Public Preview)

Today, when you write data into cloud storage, you need to compact files for optimal I/O performance. You have to worry about the right file size, how often to compact the files, how big a cluster to use, and so on. To address this class of problems, we are excited to introduce Auto Optimize with Delta Lake on Databricks. During every write to Delta tables, we automatically figure out the right file sizes and compact files so that you don’t have to worry about optimizing the storage layout. During writes, if the auto-optimize option is true, Databricks automatically determines if an optimize is required and optimizes the small files. For details, see Configure Delta Lake to control data file size.

Library Utilities generally available

Library Utilities are now generally available. This feature allows you to manipulate Python libraries from within the scope of a notebook session. See (Legacy) Library utility (dbutils.library).

Databricks Connect generally available

Databricks Connect is now generally available. This feature is a Spark client library that lets you connect your favorite IDE (IntelliJ, Eclipse, PyCharm, RStudio), notebook server (Zeppelin, Jupyter), and other custom applications to Databricks clusters and run Spark code. See Databricks Connect.

Binary file data source

The binary file data source reads binary files and converts each file into a single record that contains the raw content and metadata of the file. You can now efficiently read arbitrary files into a Spark DataFrame without visiting the content of the files. This is especially helpful if you want to use Spark to load large files, such as images, for scoring. See Binary file.

Optimized DBFS FUSE folder for deep learning workloads

Databricks now offers an optimized FUSE mount. You can have high-performance data access during training and inference without applying init scripts. Data stored under dbfs:/ml and accessible locally at file:/dbfs/ml is now backed by this optimized FUSE mount. See Load data for machine learning and deep learning.

Glue metastore (Public Preview)

Glue Catalog support is now in Public Preview. This feature lets you configure Databricks Runtime to use the AWS Glue Data Catalog as its metastore, which can serve as a drop-in replacement for an external Hive metastore. It also enables multiple Databricks workspaces to share the same metastore. For more information and limitations, see Use AWS Glue Data Catalog as a metastore (legacy).

Improvements

Disk caching

Disk caching is now preconfigured for c5d instances. However, the cache is not enabled by default. See Enable or disable the disk cache for information about how to enable it.

Upgraded R to 3.6.0

R base is upgraded from 3.5.2 to 3.6.0.
