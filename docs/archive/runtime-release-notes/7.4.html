Databricks Runtime 7.4 (unsupported)

Databricks released this image in November 2020.

The following release notes provide information about Databricks Runtime 7.4, powered by Apache Spark 3.0.

New features

In this section:

  • Delta Lake features and improvements

  • Auto Loader now supports delegating file notification resources setup to admins

  • New USAGE privilege give admins greater control over data access privileges

  • DBFS FUSE now enabled for passthrough-enabled clusters

Delta Lake features and improvements

This release provides the following Delta Lake features and improvements:

  • New API allows Delta Lake to verify that data added to a table satisfies constraints

  • New API allows you to roll back a Delta table to an older version of the table

  • New starting version allows returning only latest changes in a Delta Lake streaming source

  • Improved stability of OPTIMIZE

New API allows Delta Lake to verify that data added to a table satisfies constraints

Delta Lake now supports CHECK constraints. When supplied, Delta Lake automatically verifies that data added to a table satisfies the specified expression. To add CHECK constraints, use the ALTER TABLE ADD CONSTRAINTS command. For details, see Constraints on Databricks.

New API allows you to roll back a Delta table to an older version of the table

You can now roll back your Delta tables to older versions by using the RESTORE command:

RESTORE <table> TO VERSION AS OF n;
RESTORE <table> TO TIMESTAMP AS OF 'yyyy-MM-dd HH:mm:ss';

from delta.tables import DeltaTable
DeltaTable.forPath(spark, path).restoreToVersion(n)
DeltaTable.forPath(spark, path).restoreToTimestamp('yyyy-MM-dd')

import io.delta.tables.DeltaTable
DeltaTable.forPath(spark, path).restoreToVersion(n)
DeltaTable.forPath(spark, path).restoreToTimestamp("yyyy-MM-dd")

RESTORE creates a new commit that rolls back all changes made to your table since the version you want to restore. All existing data and metadata is restored, which includes the schema, constraints, streaming transaction IDs, COPY INTO metadata, and the table protocol version. For details, see Restore a Delta table.

New starting version allows returning only latest changes in a Delta Lake streaming source

To return only the latest changes, specify startingVersion as latest. For details, see Specify initial position.

Improved stability of OPTIMIZE

OPTIMIZE (without any partition predicates) can scale to running on tables with tens of millions of small files. Previously the Apache Spark driver could run out of memory and OPTIMIZE wouldn’t complete.OPTIMIZE now handles very large tables with tens of millions of files.

Auto Loader now supports delegating file notification resources setup to admins

A new Scala API allows admins to set up file notification resources for Auto Loader. Data engineers can now operate their Auto Loader streams with fewer permissions by delegating the initial resource setup to their admins. See Manually configure or manage file notification resources.

New USAGE privilege give admins greater control over data access privileges

To perform an action on an object in a database you must now be granted the USAGE privilege on that database in addition to the privileges required to perform the action. The USAGE privilege is granted for a database or a catalog. With the introduction of the USAGE privilege, a table owner can no longer unilaterally decide to share it with another user; the user must also have the USAGE privilege on the database containing the table.

In workspaces that have table access control enabled, the users group automatically has the USAGE privilege for the root CATALOG.

For details, see USAGE privilege.

DBFS FUSE now enabled for passthrough-enabled clusters

You can now read and write from DBFS using the FUSE mount at /dbfs/ when you are using a High Concurrency cluster enabled for credential passthrough. Regular mounts are supported. Mounts that require passthrough credentials are not supported.

Improvements

Spark SQL supports IFF and CHARINDEX as synonyms for IF and POSITION

In Databricks Runtime IF() is a synonym for CASE WHEN <cond> THEN <expr1> ELSE <expr2> END

Databricks Runtime now supports IFF() as a synonym for IF()

SELECT IFF(c1 = 1, 'Hello', 'World'), c1 FROM (VALUES (1), (2)) AS T(c1)
=> (Hello, 1)
(World, 2)

CHARINDEX is an alternate name for the POSITION function. CHARINDEX finds the position of the first occurrence of a string within another string with an optional start index.

VALUES(CHARINDEX('he', 'hello from hell', 2))
=> 12

Multiple outputs per cell enabled for Python notebooks by default

Databricks Runtime 7.1 introduced support for multiple outputs per cell in Python notebooks (and %python cells within non-Python notebooks), but you were required to enable the feature for your notebook. This feature is enabled by default in Databricks Runtime 7.4. See View multiple outputs per cell.

Autocomplete improvements for Python notebooks

Autocomplete for Python shows additional type information generated from static analysis of the code using the Jedi library. You can press the Tab key to see a list of options.

Improved display of Spark ML vectors in Spark DataFrame preview

The display format now shows labels for vector type (sparse or dense), length, indices (for sparse vectors), and values.

Other fixes

  • Fixed a pickling issue with collections.namedtuple in notebooks.

  • Fixed a pickling issue with interactively defined classes and methods.

  • Fixed a bug that caused calls to mlflow.start_run() to fail on passthrough or clusters enabled for table access control.

    • parso upgraded from 0.5.2 to 0.7.0.
