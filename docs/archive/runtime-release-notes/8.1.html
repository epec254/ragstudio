Databricks Runtime 8.1 (unsupported)

Databricks released this image in March 2021.

The following release notes provide information about Databricks Runtime 8.1, powered by Apache Spark 3.1.1.

New features and improvements

  • New SQL operators reduce complex ETL on JSON strings

  • Control visibility of table access control lists

  • New Auto Loader options enable better performance

  • Amazon S3 Select now supports CSV records with quoted delimiters

  • Track stream progress with Kinesis metrics

  • Track stream progress with Kafka metrics

  • Track stream progress with Delta Lake streaming source metrics

New SQL operators reduce complex ETL on JSON strings

Two new SQL operators—:: and :—allow you to easily extract data from JSON strings.

  • To cast your expression to a data type, use a double colon ::. For example, instead of specifying cast(col as int), you can now say col::int.

  • To extract fields from semi-structured data, use a single colon :. If you have a STRING column containing JSON data, you can extract a field from that JSON string using the syntax <json-column>:<field> which extracts the key field from your JSON <json-column> as a STRING. You can combine this with the :: syntax to cast the data to a specific type.

For example, if you have the JSON data:

{
  "store":{
    "bicycle":{
      "price":19.95,
      "color":"red"
    }
  }
}

and you create a table store_data from the data:

CREATE TABLE store_data AS SELECT
'{
   "store":{
      "bicycle":{
        "price":19.95,
        "color":"red"
      }
    }
 }' as raw

the statement SELECT raw:store.bicycle.price::double FROM store_data returns:

+--------+
| price  |
+--------+
| 19.95  |
+--------+

For details, see Query semi-structured data in Databricks.

Control visibility of table access control lists

To avoid leaking information by listing all defined permissions on a table, you can now run SHOW GRANTS [<user>] <object> statement only if you are:

  • A Databricks SQL administrator or the owner of <object>.

  • The user specified in [<user>].

New Auto Loader options enable better performance

Auto Loader now supports a new option cloudFiles.fetchParallelism to specify the parallelism to fetch messages from the queue service.

See What is Auto Loader?.

Amazon S3 Select now supports CSV records with quoted delimiters

The Amazon S3 Select data source has a new configuration option allowQuotedRecordDelimiter that indicates if CSV records can contain quoted delimiters as part of values. When the option is set to true, values with record delimiters such as "n" are interpreted as a single record. The option is disabled by default. See Amazon S3 Select for more information.

Track stream progress with Kinesis metrics

Kinesis now reports metrics about how many milliseconds falling behind the latest after every batch. You can use these metrics to track stream progress. For details on how to view the metrics, see What metrics does Kinesis report?.

Track stream progress with Kafka metrics

The Kafka streaming source now reports how many offsets a consumer is falling behind the latest after every batch. You can use these metrics to track stream progress. For details on how to view the metrics, see Retrieve Kafka metrics.

Track stream progress with Delta Lake streaming source metrics

A Delta Lake streaming source now reports how large the backlog is and how many files exist in the backlog after every batch. You can use these metrics to track stream progress. For details on how to view the metrics, see Metrics.

Apache Spark

Databricks Runtime 8.1 includes Apache Spark 3.1.1. This release includes all Spark fixes and improvements included in Databricks Runtime 8.0 (unsupported), as well as the following additional bug fixes and improvements made to Spark:

  • [SPARK-34534] Fix blockIds order when use FetchShuffleBlocks to fetch blocks

  • [SPARK-34613] [SQL] Fix view does not capture disable hint config

  • [SPARK-34392] [SQL] Support ZoneOffset +h:mm in DateTimeUtils. getZoneId

  • [SPARK-34436] [SQL] DPP support LIKE ANY/ALL expression

  • [SPARK-34497] [SQL] Fix built-in JDBC connection providers to restore JVM security context changes

  • [SPARK-34543] [SQL] Respect the spark.sql.caseSensitive config while resolving partition spec in v1 SET LOCATION

  • [SPARK-34550] [SQL] Skip InSet null value during push filter to Hive metastore

  • [SPARK-34531] [CORE] Remove Experimental API tag in PrometheusServlet
