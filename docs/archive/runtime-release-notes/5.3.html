Databricks Runtime 5.3 (unsupported)

Databricks released this image in April 2019.

The following release notes provide information about Databricks Runtime 5.3, powered by Apache Spark.

New Features

Databricks Delta

Time travel GA

Delta time travel has graduated to general availability. It adds the ability to query a snapshot of a table using a timestamp string or a version, using SQL syntax as well as DataFrameReader options for timestamp expressions.

SELECT count(*) FROM events TIMESTAMP AS OF timestamp_expression
SELECT count(*) FROM events VERSION AS OF version

Time travel has many use cases, including:

  • Re-creating analyses, reports, or outputs (for example, the output of a machine learning model), which is useful for debugging or auditing, especially in regulated industries.

  • Writing complex temporal queries.

  • Fixing mistakes in your data.

  • Providing snapshot isolation for a set of queries for fast changing tables.

See Work with Delta Lake table history.

MySQL table replication to Delta Public Preview

Stream data from a MySQL table directly into Delta for downstream consumption in Spark analytics or data science workflows. Leveraging the same strategy that MySQL uses for replication to other instances, the binlog is used to identify updates that are then processed and streamed to Databricks as follows:

  • Reads change events from the database log.

  • Streams the events to Databricks.

  • Writes in the same order to a Delta table.

  • Maintains state in case of disconnects from the source.

Private preview features

This release contains the following private preview features for Delta Lake:

  • Added an experimental command that allows Presto and Athena to directly query Delta tables.

  • Auto Optimize optimizes the file layout during individual writes to a Delta table and prevents small files from accumulating in your Delta tables to maintain lightning-speed query performance. In addition, writes and upserts into partitioned Delta tables may enjoy large speedups thanks to an adaptive shuffle introduced into writes.

If you want to participate in either preview, contact your Databricks account team.

Improvements

  • Notebook-scoped libraries:

    • Support for Extras in dbutils.library.installPyPI.

    • Support for notebook-scoped library installation for libraries in the Azure file system, including wasbs, adls, and abfss.

    • Improved error messages for notebook-scoped libraries on unsupported targets. For example: “Library utilities are not available on Databricks Runtime for Machine Learning.”

  • Databricks Advisor has additional hints to improve the performance of queries:

    • When there are many small files per partition -> Convert table to Delta and run OPTIMIZE.

    • When there is a highly selective filter -> Convert to Delta and use ZORDER BY.

    • When metadata fetches from Hive become a bottleneck -> Convert your table to Delta.

  • S3 Requester Pays: Added support for reading from Requester Pays buckets on S3.

  • Upgraded Ubuntu from 16.04.5 LTS to 16.04.6 LTS.

  • Upgraded Scala 2.11 from 2.11.8 to 2.11.12.

  • Upgraded some installed Python libraries:

    • pip: 18.1 to 19.0.3.

    • setuptools: 40.6.3 to 40.8.0

    • wheel: 0.32.3 to 0.33.1

  • Upgraded R from 3.4.4 to 3.5.2.

  • Upgraded several installed R libraries. See Installed R Libraries.

Deprecation

  • Apache Kafka 0.8 connector for Structured Streaming is no longer supported. It is included in Databricks Runtime 5.3 but will be removed from a future Databricks Runtime release.

  • Databricks ML Model Export is deprecated and will be removed in Databricks Runtime 6.0. Use MLeap for importing and exporting models instead.

  • Removed spark.databricks.pyspark.enableExecutorFsPerms as a configuration option.

Bug fixes

  • Fixed cluster-wide Python Egg library installation for clusters enabled for table ACLs.

  • Improved Python command cancellation by fixing the situation in which cancellation is called before the command is executed.

  • Fixed a bug in table ACLs: now when you list objects in a database or catalog, you see only the objects that you have permission to see.
