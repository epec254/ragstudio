Databricks Runtime 5.5 LTS (unsupported)

Databricks released this image in July 2019. It was declared Long Term Support (LTS) in August 2019. Support ended on July 27, 2021. Databricks Runtime 5.5 Extended Support (unsupported) was released on July 8, 2021 and extends 5.5 support through December 2021. It uses Ubuntu 18.04.5 LTS instead of the deprecated Ubuntu 16.04.6 LTS distribution used in the original Databricks Runtime 5.5 LTS. Ubuntu 16.04.6 LTS support ceased on April 1, 2021.

The following release notes provide information about Databricks Runtime 5.5, powered by Apache Spark.

New features

Delta Lake on Databricks Auto Optimize

Today, when you write data into cloud storage, you need to compact files for optimal I/O performance. You have to worry about the right file size, how often to compact the files, how big a cluster to use, and so on. To address this class of problems, we are excited to announce the general availability of Auto Optimize with Delta Lake on Databricks. During every write to Delta tables, we automatically figure out the right file sizes and compact files so that you don’t have to worry about optimizing the storage layout. During writes, if the auto-optimize option is true, Databricks automatically determines if an optimize is required and optimizes the small files. For details, see Configure Delta Lake to control data file size.

Delta Lake on Databricks improved min, max, and count aggregation query performance

The performance of min, max, and count aggregation queries for Delta Lake on Databricks has been significantly improved by reducing the amount of data that’s read. These queries are now run using statistics and partition values in the metadata, rather than scanning the data.

Presto and Athena support for Delta tables on AWS S3 (Public Preview)

You can now query Delta tables from external tools such as Presto and Athena. Presto and Athena support reading from external tables when the list of data files to process is read from a manifest file, which is a text file containing the list of data files to read for querying a table. When an external table is defined in the Hive metastore using manifest files, Presto and Athena use the list of files in the manifest rather than finding the files by directory listing.

Glue Catalog as Databricks metastore

Glue Catalog support is generally available. This feature lets you configure Databricks Runtime to use the AWS Glue Data Catalog as its metastore, which can serve as a drop-in replacement for an external Hive metastore. It also enables multiple Databricks workspaces to share the same metastore. For more information and limitations, see Use AWS Glue Data Catalog as a metastore (legacy).

Preview

Credential passthrough for Glue Catalog is in Private Preview. Contact support if you’d like to try this out.

Faster model inference pipelines with improved binary file data source and scalar iterator pandas UDF (Public Preview)

Machine learning tasks, especially in the image and video domain, often have to operate on a large number of files. In Databricks Runtime 5.4, we already made available the binary file data source to help ETL arbitrary files such as images, into Spark tables. In Databricks Runtime 5.5, we have added an option, recursiveFileLookup, to load files recursively from nested input directories. See Binary file.

The binary file data source enables you to run model inference tasks in parallel from Spark tables using a scalar pandas UDF. However, you might have to initialize the model for every record batch, which introduces overhead. In Databricks Runtime 5.5, we backported a new pandas UDF type called “scalar iterator” from Apache Spark master. With it you can initialize a model only once and apply the model to many input batches, which can result in a 2-3x speedup for models like ResNet50. See Series to scalar UDF.

Secrets API in R notebooks

The Secrets API lets you inject secrets into notebooks without hardcoding them. This API is now available in R notebooks in addition to existing support for Python and Scala notebooks. You can use the dbutils.secrets.get function to obtain secrets. Secrets are redacted before printing to a notebook cell.

Improvements

  • Support for running Delta Lake SQL operations in Python foreachBatch: We have fixed the known limitation of not being able to write to Delta tables from inside foreachBatch of a Structured Streaming query defined in Python. This is useful in a common Python streaming workloads; for example, Writing streaming aggregates in update mode using MERGE and foreachBatch.

  • Performance of Delta tables stored in Azure Data Lake Gen2: The check for the latest version of a Delta table on ADLS Gen2 now only checks the end of the transaction log, rather than listing all available versions. This optimization makes UPDATE a constant time operation, significantly improving the latency.

  • Scalability of Optimize ZORDER BY: Z-Ordering on very large Delta tables now uses smaller units of work that are governed by advanced admission control. This feature improves the stability of this operation without sacrificing cluster utilization.

  • Improved performance of DML commands on tables with large number of columns: We now perform better column pruning when scanning for matching data in UPDATE, DELETE, and MERGE commands.

  • Support for VNet + Service Endpoints setup in Spark - Synapse Analytics connector: We added ADL Gen2 paths to an allow list as temp data locations (.option("tempDir", "abfss://...")) and added a new option called useAzureMSI, to be used instead of forward_spark_azure_storage_credentials in case your Synapse Analytics is configured to authenticate via Managed Identities to your V2 storage account.

  • Automatic invalidation for disk caching: Disk caching now automatically detects files that have been modified or overwritten after being cached. Any stale entries are automatically invalidated and evicted from the cache. See Optimize performance with caching on Databricks.

  • Upgraded Python library wheel from 0.33.3 to 0.33.4.

  • Upgraded R library nlme from 3.1-139 to 3.1-140.

Bug fixes

  • Fixed a bug that the S3-SQS/ABS-AQS connector may hit a corrupt Delta table error when a file gets removed and re-added.

  • Fixed cancellation for R commands that do not run Spark jobs. Previously, R commands that do not run Spark jobs could be cancelled, but the notebooks’ state would be lost; commands can now be cancelled without losing notebook state.

  • Dropping or moving a managed table now invalidates the cached Delta Lake log.

  • Fixed a bug that writing Delta Lake checkpoint may fail due to FileAlreadyExistsException.

  • Scala REPL now sets the proper -target:jvm-1.8 flag to support calling Java methods that use Java 8 features.
